script标签的书写方法深挖

要点

  1. script标签的type属性不是必须的，默认缺省就是text/javascript
  2. script标签的language属性完全无用（asp时代微软似乎使用该属性来标记服务端语言是vb还是c#），不要画蛇添足

  3. 动态创建的script标签必须要指定type=’text/javascript’，否则JS不会执行

  4. 动态创建的script追加动作是异步的，并不会立刻取得script运行结果，如果要等待加载完成需要监听完成事件

  5. 使用非标准或者比较新的属性需要格外注意，不要使代码逻辑依赖于这些特性。如defer/async属性
  6. 使用script.onerror来监听脚本执行失败的情况（语法错误，初始化运行时错误等都会触发）
  7. 监听script的完成事件比较复杂。

jQuery/Zepto选择器的.text()和.html()方法

现状：大多数开发同学会混淆两者并乱用，不清楚何时用哪个
详解：.text()方法用于获取和设置文本内容，.html()方法用户获取和设置HTML内容，当要设置或获取的内容仅仅为文本时，两者行为完全相同，但要操作的文本内容是HTML时，行为有着本质区别。

总结：

  1. 根据实际需要选择使用哪个方法，如能断定内容为纯文本请使用text()方法。仅当确实需要渲染HTML时才用html()方法
  2. 从安全角度，text()方法比html()方法更安全，无注入风险。
  3. 严格意义上，html()方法不符合CSP规范，直接将字符串解析为DOM节点
  4. 业务需要确实要使用.html()方法渲染动态内容时，必须做安全检查，避免恶意代码注入
  5. .text()和.html()获取值可能存在代码缩进（空格和TAB），如有需要可以使用$.trim()来剔除

数组与对象深挖

要点：

  1. 数组对象仅有concat/reverse/slice/splice为标准API，而且绝对完全兼容
  2. 数组对象请勿使用indexOf、lastIndexOf、map、every、forEach等非标准API，不仅兼容性存在问题，而且效率不一定高，反而不如自己实现

  3. 遍历数组请将.length缓存到变量

  4. 遍历数组请勿使用此写法

  5. 遍历key-value型对象必须使用hasOwnProperty()来过滤遍历结果。

  6. 不论是数组或对象，在遍历操作时不要改变被遍历的变量结构，如增删元素，增删key值等（虽然你可以这么做）,对于元素自身及子成员的修改是绝对安全的。

关于Prototype的使用

要点：

  1. 不要乱用Prototype。不要轻易在Object/Array/Function等对象上追加prototype（虽然我们已有某些库这么做了）容易产生歧义冲突，在使用for～in遍历时很容易引发问题。
  2. 自定义的prototype成员会在for～in循环中出现，请根据实际情况使用hasOwnProperty()来过滤遍历结果。

不严谨的写法：

  1. 对象的__proto成员，用途是获取当前实例的原型对象。非标准实现，存在兼容性问题，请不要使用
  2. 原则上不要轻易重写已存在的prototype方法。但可以在单个实例中覆写该方法
  3. prototype上定义静态对象变量，会造成所有对象共用，而不是分别创建实例，请在构造方法中分配实例
